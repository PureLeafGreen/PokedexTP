Exercice intégrateur Git

On vous fourni le code de base d'un Pokédex simple, qui vous permettra de documenter les pokémons que vous croiserez lors de vos aventures au cégep. Mais avant de partir pour tous les attraper, vous devrez apporter 3 séries de modifications à l'application.

Pour simuler une situation plus réaliste, on vous demande de faire vos changements dans trois branches séparées:
    bugfix: Un bug s'est glissé qui cause un crash lorsque l'on essaie d'afficher l'évolution d'un pokémon qui n'en a pas. Vous devez le corriger puis ajouter une fonctionnalité similaire pour leur forme précédente.
    feat1 : On souhaite ajouter une méthode pour rechercher un pokémon selon son nom, et une autre pour rechercher un ensemble de pokémons selon leur type (c-à-d, tous les pokémons d'un type donné). Ajoutez donc cette fonctionnalité.
    feat2 : On veut pouvoir obtenir dans la classe Pokemon la liste combinée de toutes ses faiblesses; si le pokémon a plus d'un type avec des faiblesses communes, elles ne doivent pas être listées en double. Affichez ces faiblesses à l'utilisateur en mots.

Ensuite, vous devrez combiner (merge) vos trois branches en une (master).
Dans cette branche, vous devriez avoir un menu complet qui supporte toutes les fonctionnalités ci-haut.

Bon travail!
